1. General-purpose API collections
Repository	Best use	Assessment
public-apis/public-apis	Discovering free APIs across many categories	The best-known starting point. It classifies APIs by authentication, HTTPS, CORS, and category. As of July 11, 2026, it had roughly 448,000 stars and had received code updates in June 2026. Its weakness is that individual entries may become paid, discontinued, or outdated between checks.
public-api-lists/public-api-lists	A more structured, actively validated free-API directory	One of the strongest alternatives to public-apis. Its associated catalog currently advertises more than 800 APIs in 48 categories and exposes JSON endpoints, making it easier to use programmatically.
APIs-guru/openapi-directory	OpenAPI definitions, SDK generation, testing, agents and automation	More valuable than a simple link list when you need machine-readable API schemas. It contains thousands of OpenAPI descriptions and more than 100,000 documented endpoints, with automated conversion and validation processes.
ripienaar/free-for-dev	Free tiers for developers	Covers databases, cloud hosting, authentication, monitoring, CI/CD, AI services and APIs. It is not exclusively an API directory, but it is extremely useful when building a real application on a limited budget.
sindresorhus/awesome	Finding specialist collections	A directory of curated “awesome lists.” Use it when you need APIs or datasets for a particular subject such as healthcare, finance, maps, cybersecurity, NLP or computer vision.
tools-collection/apis-collection	API documentation, SDK and specification discovery	Claims more than 1,200 APIs and includes documentation, specifications, SDKs and related libraries. It is a useful secondary source after checking the three repositories above.
keploy/public-apis-collection	Beginner projects and API experimentation	A curated collection that attempts to describe API usability and complexity. Useful for project ideas, but not as comprehensive as the main directories.
alexandresanlim/public-apis-no-auth-only	APIs that work without keys	A filtered collection focused on no-authentication APIs. Convenient for prototypes, coding practice and demonstrations, although it may inherit outdated records from broader API lists.

Government APIs and open-government data
Worldwide resources
Repository	What it contains
okfn/dataportals.org	A community-maintained catalog of national, regional and municipal open-data portals around the world. It helps locate official portals rather than listing every government endpoint individually.
github/government.github.com	A catalog of government organizations and civic-technology groups using GitHub. This is useful for finding repositories maintained directly by departments, cities and public agencies.
i-dot-ai/awesome-gov-datasets	Curated United Kingdom government datasets, maintained by the UK government’s Incubator for Artificial Intelligence.
okhosting/awesome-open-data	Open-data sources covering government, demographics, economics, transport, science and international organizations.
jdorfman/awesome-json-datasets	Public JSON datasets and endpoints, including government and economic data. Useful for quick prototypes because the sources are already JSON-oriented.
rnckp/awesome-ogd-switzerland	Swiss government portals, APIs, tools and related international open-government resources.
India-specific resources
Official sources—use these first
Data.gov.in APIs — India’s official Open Government Data API directory.
API Setu — Government platform for publishing and consuming APIs from ministries, departments and other organizations.
GitHub repositories
Repository	Assessment
addypy/datagovindia	Python client for searching, discovering and downloading resources from India’s OGD platform. This is one of the most practical choices for a Python data project.
econabhishek/datagovindia	R interface for discovering API information and querying data.gov.in resources as tidy datasets.
ctrlaltvikas/Indian-government-API-List	A useful historical collection of Indian government API links. However, its documented update date is February 2020, so every endpoint must be rechecked against Data.gov.in or API Setu before use.

For an Indian-government project, I would use Data.gov.in/API Setu as the source of truth, then use addypy/datagovindia or econabhishek/datagovindia to access the data programmatically.
